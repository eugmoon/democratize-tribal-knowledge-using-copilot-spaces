# OctoAcme Project Management Docs

Welcome — this folder contains OctoAcme's project management guidance, templates, and checklists to help teams deliver work consistently and onboard new teammates quickly.

## Brief overview of our process
OctoAcme follows a customer-first, iterative delivery model with clear ownership and measurable outcomes. Projects move through a lightweight lifecycle: Initiation (one-pager + decision gate), Planning (backlog, estimates, Definition of Done), Execution & Tracking (work on a project board, small PRs, CI, demos), Release & Deployment (staged rollout and rollback plans), and Close & Retrospective (blameless reviews and tracked action items). Roles (Product Manager, Project Manager, Developers, QA) each have explicit responsibilities for outcomes, delivery, and communications.

## Key practices
- Single source of truth: keep the Project Charter and process artifacts in the project repo (docs/ or .copilot/).
- Workflows: project board columns Backlog → Ready → In Progress → In Review → QA → Done, small PRs with issue links, automated tests and security scans in CI, and at least one approval before merging.
- Communication cadence: daily standups, weekly PM+PdM syncs, sprint demos, monthly stakeholder updates, and defined escalation paths for blockers or incidents.
- Quality & improvement: unit/integration/e2e smoke tests, monitoring dashboards, and retrospectives that produce 2–3 prioritized action items tracked in the backlog.
